Codename: GamePI

Websites about video games are in abundance, good data from said games on these sites is a whole other story.

What is Codename: GamePI

What would a centralized game data api look like? How useful would it be to supply websites with information about the internal data of games? I dunno, but as a project I decided to find out. GamePI plans to offer detailed information about various video games, including but not limited to: item stats, character stats, weapon stats, map data, and more. Data dumps from each game will be automated via scripts that parse the game files and extract the relevant information, with version watches built in to automatically update on releases. This data will then be stored in a database and made accessible via both GraphQL and REST APIs via GraphQL2REST.

Sounds neat, can I use it?

Not yet, I have 964 Unreal Engine games in my Steam library and i'm about halfway through analyzing the data and building automation scripts. Once I reach 1000 games in the database, I will launch version 1.0 with the following features:

  • Browsable Game Pages with information on the data collected.
  • Full featured API (*Rate Limited).
  • Tools to submit your game data to the site.
  • Build Pipeline Scripts for Unreal, Godot, and GameMaker so developers can provide updated data should they choose.
  • A sense of completion for a simple but technically complex project.
