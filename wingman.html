IT-Wingman

The IT Wingman project aimed to replace an existing solution and advance the equipment tracking and management within a particular Air Force Medical Group IT Deparment. It is designed to be a comprehensive intranet based application empowering IT personnel in their daily tasks, providing them with the tools to efficiently handle equipment checkouts, returns, repairs, imaging, and software installations. The application prioritizes user-friendliness, reliability, and data security.

Goals and Objectives

The primary goals of the IT Wingman project are:

  • Streamline Processes: Automate and simplify routine tasks like checkouts, returns, and inventory updates, reducing manual effort and potential errors.
  • Enhanced Visibility: Provide a centralized, up-to-date view of equipment inventory, location, status, and usage history.
  • Improved Efficiency: Facilitate quicker repairs and imaging processes to reduce equipment downtime and ensure resources are available when needed.
  • Accountability: Establish a clear audit trail of equipment usage and maintenance actions, fostering responsibility and transparency.
  • Data-Driven Insights: Generate reports and visualizations to help identify trends, optimize resource allocation, and make informed decisions.

Scope and Deliverables

The project scope encompasses:

  • Development: Creating a Rust-based intranet based application with a modern, intuitive graphical user interface (GUI) using the Iced framework.
  • Database: Implement a robust and scalable MS SQL Server database schema to store equipment, customers, transactions, and related data.
  • Integration: Seamless integration with Active Directory for user authentication, role-based access control (RBAC), and user provisioning.
  • Core Features:
    • Equipment check-out/check-in with barcode scanning and digital signature of PDF forms.
    • Repair and imaging tracking with detailed logs.
    • Software installation and license management.
    • User/customer management and lookup with Active Directory integration.
    • Comprehensive reporting with filtering, sorting, and export options.
    • Customizable home screen with widgets for quick access to information and tools.

The project scope explicitly excludes:

  • Integration with external IT systems (like ServiceNow) in the initial release.

System Architecture

As the Fed pushes for Rust, it was determined the application would be more likely to pass the necessary security reviews if built in Rust. The Iced GUI library was chosen for the front-end due to its simplicity and WASM Compataility layer.

User Interface Design

The IT Wingman application's user interface (UI) will be intuitive, efficient, and visually appealing, adhering to modern design principles. The goal is to provide a seamless user experience for all roles, from service technicians, and system administrators, to cyber security and IT Administrators, while ensuring each role has access to the specific features and data they need.
